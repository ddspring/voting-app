## 🚀 快速部署

### 方法一：Vercel 部署（推荐，永久免费）

1. **Fork 本项目到你的 GitHub**

2. **访问 [vercel.com](https://vercel.com)，用 GitHub 登录**

3. **点击 "Import Project"**
   - 选择你 fork 的仓库
   - 点击 Deploy

4. **获取永久链接**
   - 部署完成后会得到类似 `https://voting-app-xxx.vercel.app` 的链接
   - 可以分享给任何人使用

### 方法二：本地运行

```bash
cd voting-app
npm install
npm start
```

访问 `http://localhost:3847`

局域网访问：`http://你的IP:3847`（手机和电脑同一WiFi）

## 📱 使用流程

### 1️⃣ 主持人创建投票

- 打开链接
- 输入投票议题
- 选择投票时长
- 输入管理员昵称
- 点击"创建投票"

### 2️⃣ 分享投票链接

创建成功后，复制"投票链接"发给群友

### 3️⃣ 参与者投票

- 打开投票链接
- 输入真实姓名
- 选择：✅ 赞成 / ❌ 不赞成 / 🔶 有条件赞成
- 如选"有条件赞成"，需填写条件说明
- 点击提交

### 4️⃣ 主持人查看结果

用"管理链接"实时查看：
- 投票统计（总票数、赞成/不赞成/有条件各多少）
- 每个人的投票详情
- 有条件赞成的具体条件

## ❓ 常见问题

**Q: 投票数据会丢失吗？**
A: 本地运行时数据保存在 `data/voting-data.json`，Vercel 部署时数据保存在 `/tmp`（服务重启会清空，适合单次会议使用）

**Q: 可以多少人投票？**
A: 没有硬性限制，建议 20 人以内

**Q: 投票可以修改吗？**
A: 目前不支持修改，每人只能投一次
